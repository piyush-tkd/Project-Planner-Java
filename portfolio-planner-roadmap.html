Friction & Gaps I Noticed

Sidebar has 45+ items — confirmed in the UI.Even with collapsible groups (Home, Portfolio, People, Calendar, Delivery, Engineering, Simulations, Admin), full-access users face a wall of options. The "Projects" link lives under no group header — it's accessed via breadcrumb "Data Entry" which is confusing. Need: search/command palette, pinnable favorites, recent pages.

Capacity Hub tabs are "View Full Report" stubs.Several capacity sub-views (Utilization, Hiring Forecast, etc.) show a description card with a "View Full Report" button rather than inline content. This adds unnecessary click depth. The data should render inline with the tab selection.
Timeline Simulator uses number spinners.Each project has Start Month and Duration spinners — adjusting 20+ projects one by one is tedious. A drag-on-Gantt interaction (like Planview) would make this 10x more usable. No visual preview until you click "Simulate."

Financial Intelligence: 3.2% capacity utilization.Either the data is early-month or the metric needs context. A "3.2% — Below expected" badge without benchmarks or trends looks alarming. Add sparkline trend, target line, and month-over-month delta.

The Moat Strategy: Why No One Can Catch You

Portfolio Planner's defensibility isn't any single feature — it's the combination. Here are the 5 moat layers and how each phase deepens them.

1. Data Gravity

Jira sync (20+ entities), resource data, financials, Git analytics, and AI conversation history create a data flywheel. The more you use it, the harder it is to leave. v14 adds AI proactive insights that get smarter over time.

Competitors can't replicate: Your NLP self-learning system improves with every query. Jellyfish has analytics but no resource/project context. Planview has portfolio but no Jira depth.

2. Cross-Domain Intelligence

No tool connects portfolio health ↔ resource capacity ↔ Jira delivery ↔ financial intelligence ↔ Git metrics in a single query. Your AI assistant can answer "Which POD is most cost-effective per story point?" because it has all the data.

To replicate: Competitor would need Jira + Tempo + Jellyfish + Planview + custom glue code. That's $80–200/user/month and 6 months of integration work.

3. AI-Native Architecture

pgvector embeddings, self-learning patterns, SSE streaming, tool catalog routing — this isn't a chatbot bolted on. v14 expands to proactive insights and AI impact measurement. v15 adds SPACE framework. Each version makes the AI more valuable.

Competitors adding AI: Wrike (surface-level), Smartsheet (formula assist), Monday (automations). None have semantic search over portfolio data or self-learning from user feedback.

4. Scenario Simulation Engine

Named scenarios with isolated overrides, timeline simulation, and what-if analysis — without corrupting live data. v14 upgrades this to a visual drag-on-Gantt experience that makes planning feel intuitive.

Planview has this at $45+/user. No mid-market tool (Monday, Asana, Wrike, Shortcut) offers it. Your simulation engine + AI = "show me what happens if we delay Project X and reassign 2 developers."

5. Engineering-Specific Financial Intelligence

CapEx/OpEx at the Jira issue level. Cost per story point per POD. Fully loaded engineer cost attribution. Software capitalization automation (v14). This is CFO-grade intelligence built for engineering orgs.

Jellyfish charges premium for software capitalization. Kantata does PS financials but not engineering. Tempo does cost tracking but not portfolio-level intelligence. You're the only one that connects all three.

UI / Design System Recommendations

How to make Portfolio Planner visually rich, modern, and usable — from design system choice to specific component upgrades.

Current Design State

Portfolio Planner currently uses Mantine v7 as its component library with a custom theme (deepBlue, aqua, legacyBlue color tokens). The foundation is solid — Mantine is a well-maintained React library with accessibility built in. However, the app's visual richness doesn't yet match the sophistication of its data model. Here's what to keep, what to upgrade, and what to add.

Design System: Keep Mantine — But Layer These On Top

Current Mantine v7

Solid component primitives (120+ components), dark mode, good TypeScript support, accessible by default. Your custom theme tokens (deepBlue/aqua) already give it identity. Mantine's DataTable, Tabs, Modals, and Forms power the core experience.

Keep Mantine as the foundation — switching to shadcn/ui or Chakra would be 3+ months of rewrite with no user-facing value.

Recommended Layered Enrichment Strategy

Keep Mantine for primitives. Add specialized libraries for the areas where Mantine falls short: data visualization, drag interactions, animation, and data grids. This gets you 80% of the visual richness of tools like Linear and Monday without a rewrite.

Cost: ~3 weeks of design system investment across v13–v14. ROI: dramatically richer UI that competes with best-in-class PM tools.

Recommended Libraries to Add

Recharts → Replace Chart.js

React-native charting that plays perfectly with Mantine's theme tokens. Declarative API, responsive, animated by default. Use for: utilization sparklines, capacity trend charts, DORA metric gauges, financial trend lines. Chart.js in React always feels like a wrapper — Recharts feels native.

npm: recharts · React-native · 3.5M weekly downloads

@dnd-kit/core — Drag & Drop

The modern React DnD library. Use for: Kanban board (Project Board view), sidebar reordering, dashboard widget rearrangement, Sprint Planner drag-to-assign, Gantt bar dragging. Accessible, touch-friendly, and composable. This single library unlocks 5+ features in the roadmap.

npm: @dnd-kit/core + @dnd-kit/sortable · React 18 compatible

Framer Motion — Micro-Animations

Add life to the UI: page transitions, card expand/collapse, sidebar toggle, toast notifications, modal entrances, loading skeletons. The difference between "functional" and "feels premium" is 200ms of thoughtful animation. Linear and Notion both use Framer Motion extensively.

npm: framer-motion · 5M weekly downloads · pairs with Mantine

Visual Polish Playbook — 10 High-Impact Changes

These are CSS/component-level changes that dramatically improve perceived quality without architectural work.

1. Loading Skeletons Everywhere

Replace blank-while-loading with shimmer skeletons. Mantine has Skeleton built in. Apply to: every table, every card grid, every chart area. This alone fixes the "20 pages render blank" problem for users with slow connections.

2. Status Badges with Color + Icon

Current project status badges are text-only. Add colored dots + subtle background tint: 🟢 Active, 🟡 At Risk, 🔴 Blocked, ⚪ On Hold. Use Mantine Badge with leftSection for consistent treatment across all tables and cards.

3. Avatar Stacks for Team Visibility

Replace text lists of owners with stacked avatar circles (initials or photos). Show on: project cards, POD headers, sprint planner cards. Mantine's Avatar.Group with tooltip on hover for names. Makes team composition instantly scannable.

4. Sparkline Trends on Every KPI

Every number that changes over time should have a tiny trend line next to it. Utilization: ↗ 85% (with 6-month sparkline). Budget burn: ↘ $1.2M. Use Recharts Sparkline or a tiny inline SVG. This transforms dashboards from snapshots to stories.

5. Page Transitions & Content Fade-In

Add framer-motion AnimatePresence to route changes. Cards fade up on mount. Tab content cross-fades. Sidebar collapse animates. Keep durations short (150–250ms). The goal isn't flashy — it's removing the "jarring content swap" feel.

6. Contextual Empty States with Illustration

Each empty state gets: a subtle illustration (use Undraw or custom SVG), a 1-line explanation, and a primary CTA button. "No projects yet → Create your first project." "Jira not connected → Set up Jira integration." Template this as a reusable EmptyState component.

7. Breadcrumb + Page Header Consistency

Standardize every page to: Breadcrumb → Page title (h1) → Description (muted subtitle) → Action buttons (right-aligned). Currently some pages have this, others don't. Create a PageHeader component with consistent spacing, divider, and action slot.

8. Sidebar Visual Refresh

Add: active page indicator (left accent border, not just highlight), section divider lines between nav groups, collapse memory (persist open/closed sections), notification badges on items with alerts (Inbox badge count, Support Queue count).

9. Data Density Toggle

Add a compact/comfortable/spacious toggle for data-heavy pages (Resources, Availability, Projects). Compact mode tightens row height and reduces padding — power users love this. Comfortable is default. Persist preference in user settings.

10. Dark/Light Mode Toggle

Mantine has built-in dark mode support and your CSS variables are already structured for it. Add a theme toggle in the user menu. The deepBlue/aqua palette works in both modes. Test the 8 most-used pages in light mode and adjust contrast ratios.

Design System Architecture — How to Organize It

Token Layer (already started)

Your Mantine theme defines deepBlue, aqua, legacyBlue as custom colors with 10 shade scales each. Extend this with semantic tokens: --success, --warning, --danger, --info mapped to your palette. Add spacing scale tokens (4/8/12/16/24/32/48) for consistent layout.

File: src/theme.ts → extend MantineThemeOverride

Component Library (build incrementally)

Create a src/components/ui/ folder with shared components that wrap Mantine primitives with your design decisions baked in: PageHeader, EmptyState, StatusBadge, KpiCard, SparklineMetric, AvatarStack, DataTable (TanStack wrapper). Each component = 1 file, typed props, Storybook story.

Pattern: Wrapper components, not a fork of Mantine

Storybook for Component Docs

Add Storybook to document your shared components. Each component gets: visual preview, prop documentation, variants (sizes, states), dark mode preview, and usage examples. This becomes the living design system for your team — faster than Figma for developers.

npm: @storybook/react-vite · Vite-native, fast HMR

Icon System — Lucide or Tabler

Mantine uses Tabler Icons by default. Stick with it for consistency — 4,500+ icons, MIT licensed, designed for Mantine. Avoid mixing icon libraries. Create an Icon wrapper that enforces consistent sizing (16/20/24px) and color inheritance.

npm: @tabler/icons-react (already in Mantine's ecosystem)

Competitive Visual Benchmarks — Where to Aim

Linear — The Gold Standard

What to steal: Cmd+K command palette, keyboard-first navigation, subtle animations on every interaction, monochromatic icon system, ultra-clean card layouts, smooth page transitions. Linear proves that enterprise tools can feel like consumer apps.

Priority: Adopt Cmd+K, animation philosophy, card design language

Monday.com — Rich Data Visualization

What to steal: Color-coded status columns, timeline bars inline with data rows, drag-to-assign interactions, dashboard widget customization (drag grid), celebration animations on completion. Monday proves that color = information density.

Priority: Adopt inline timeline bars in Project table, celebration micro-interactions

Notion — Flexible Layout

What to steal: Slash-command content insertion, toggle blocks for progressive disclosure, database views (table/board/calendar/gallery toggles), breadcrumb navigation, minimal chrome. Notion proves that flexibility > feature count.

Priority: Adopt view-switching patterns, progressive disclosure for complex pages

Figma — Collaboration UX

What to steal: Multiplayer cursors, real-time presence indicators, inline commenting, version history with named snapshots. Figma proves that collaboration features make tools stickier than feature features.

Priority: v15 — real-time presence, inline commenting on portfolio items

TL;DR — The Design System Strategy

Keep Mantine v7 as the foundation — it's the right choice and switching would cost months with zero user value.

Add 8 specialized libraries incrementally across v13–v15: cmdk, framer-motion, Recharts, @dnd-kit, TanStack Table, React Flow, @visx, and a Gantt lib. Total bundle cost: ~150KB gzipped. Each unlocks multiple roadmap features.

Build 6 shared components (PageHeader, EmptyState, StatusBadge, KpiCard, SparklineMetric, AvatarStack) that enforce consistency. Document in Storybook starting v14.

Apply 10 visual polish changes — skeletons, animations, badges, sparklines, transitions — that transform the app from "functional internal tool" to "premium SaaS product." Most are CSS/component-level, not architectural.

The goal: Make Portfolio Planner feel like Linear built a PPM tool. The data model is already world-class — the UI just needs to match.
