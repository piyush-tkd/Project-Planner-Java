The Mock-First Rule

Add this to EVERY Cowork prompt that involves UI changes. It forces Claude to show you a preview before writing production code.

Add This Block to Every UI Prompt

MOCK-FIRST RULE:
Before writing ANY production code, you MUST:

1. Create a standalone HTML mock file at: /tmp/mock-preview.html
   - Show EXACTLY what the component/page will look like
   - Use inline CSS that matches our theme (dark background #0f1117, text #e2e4eb, accent #2DCCD3)
   - Include both the "before" (current state) and "after" (your proposed change) side by side
   - If it's a component, show it in all its variants/states

2. Tell me: "Mock ready for review at /tmp/mock-preview.html"

3. WAIT for my approval before writing any React/Java code

4. If I say "approved" — proceed with implementation
   If I say "change X" — update the mock and show me again
   If I say "rejected" — stop and discuss

This applies to: new components, page layout changes, empty states,
loading states, any visual change. It does NOT apply to backend-only
changes (APIs, migrations, services).

The Workflow

You paste prompt
→
Claude creates HTML mock
→
You review the mock
→
Approve / Request changes
→
Claude writes production code
→
Build + verify

Example: How It Works in Practice

You say: "Add empty states to the 5 Delivery group pages. [MOCK-FIRST RULE block]"

Claude creates: An HTML file showing all 5 empty states side by side — icon, title, description, CTA button for each page.

You review: "The Support Queue icon should be a headset, not a wrench. And make the CTA button say 'Configure Boards' instead of 'Go to Settings'."

Claude updates: New mock with your feedback applied.

You say: "Approved. Proceed."

Claude: Now writes the actual React code, matching the approved mock exactly.

Why This Works

Without the mock step, Cowork makes 40 file changes and you discover the UI doesn't match your expectations AFTER the code is written. Fixing it means another round of changes that might break something else. The mock costs 30 seconds to create and saves hours of rework.
